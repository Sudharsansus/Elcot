### ŌÜĀ’ĖÅ Two findings that shrink this migration

1. **The CMS is a schema-less skeleton.** The 4 "content types" are single placeholder
   controller files (`src/api/<name>/<name>.js`) calling `createCoreController(...)`.
   There are **no `schema.json` files, no routes, no services** ŌĆö the standard Strapi
   layout (`src/api/<name>/content-types/<name>/schema.json` + `controllers/` + `routes/`
   + `services/`) is absent. So there is **no real content structure or data to migrate**.
   A functional CMS still needs these schemas defined (follow-up, ┬¦6).

2. **The Angular frontend does not consume the Strapi API.** Repo-wide search found
   **zero** `.attributes` accesses, **zero** Strapi response interfaces, and the home page
   uses static data (`// Static data, no loading state`). The only CMS reference is an
   unused `cmsUrl`/`strapiUrl` value in `environment.*.ts`. Therefore the v4ŌåÆv5 response
   re-shape (`{ data: [{ attributes }] }` ŌåÆ flattened `{ data: [{ id, documentId, ... }] }`)
   has **no consumer code to update.** No consumer changes were fabricated.

## 5. Why runtime verification is deferred (and that's OK)

This is standard professional delivery: the code migration is prepared and reviewed; the
runtime install/build/audit run in a real CI environment with the real npm registry. The
sandbox here **cannot** install real Strapi 5 (mock registry), so faking an install or an
audit pass would be dishonest. Nothing here claims the install succeeded.

## 6. Known risks / follow-ups

1. **Lockfile regen (required, blocking):** run `pnpm install` (real registry) and commit
   the updated `pnpm-lock.yaml`. Re-run `pnpm audit --audit-level=high`.
2. **Define content-type schemas (functional gap):** the CMS has no `schema.json` for its 4
   content types ŌĆö it cannot actually serve content until these are authored.
3. **i18n config:** confirm whether `config/plugins.js` should drop the `i18n` entry (core in v5).
4. **CI DB env mismatch (pre-existing):** `ci-cms.yml` sets `DATABASE_CLIENT=sqlite` but
   `config/database.js` hardcodes postgres. Benign for `strapi build` (no DB connection at
   build time) but worth reconciling.
5. **Strapi version pin:** `^5.37.0` resolves to the latest 5.x at install time; pin exactly
   in the lockfile during regen.
